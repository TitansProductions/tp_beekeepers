# What has been updated?

## Changelog - (Added, Fixed, Removed):

1. Added a new system for allowing owners of apiaries to be able to move those apiaries to other locations.

## Modified Files:

- client/**.lua (All)
- server/**.lua (All)
- server/tp-server_escrow_ignore.lua (add new file)
- locales.lua
- fxmanifest.lua

## Notes

1. Add the new option on `config.lua` file below `Config.RemoveApiaryPlacementByOwners`:

```lua
-- Set to true if you want the apiary owners to be able to move an apiary placement to somewhere else.
Config.MoveApiaryPlacementByOwners = false -- version 2.0.6
```

2. Add new locales on `locales.lua` file:

```lua
    ['APIARY_MENU_MOVE_APIARY_POSITION_TITLE'] = "Move Apiary Location", -- 2.0.6
    ['APIARY_MENU_MOVE_APIARY_POSITION_DESCRIPTION'] = "", -- 2.0.6

    ['APIARY_NOT_ALLOWED_TO_BE_PLACED'] = { -- 2.0.6
        title = "Apiary", 
        message = "You are not authorized to place the specified apiary - no permissions.",
        duration = 5,
        align = 'left',
    },
```

3. Add the new server side file on `server` which is called `tp-server_escrow_ignore.lua`

*Unfortunately, i cannot test the metadata support on VORP or RSG, but as far i am aware, the VORP Inventories that are behind many versions, cannot use this feature because those kind of functions i use, do not exist on the old versions.*

*If you find any issues for the item removal, please open a ticket because i used the documentation to remove the items.*
